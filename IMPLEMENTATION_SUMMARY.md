## 🎯 הצעדים הבאים

### שלב 1: בדיקה (היום)
1. הפעלת `yarn dev`
2. בדיקה של כל העמודים
3. בדיקה של טופס הזמנה

### שלב 2: תמונות (זה שבוע)
1. העלאת תמונות האמנית
2. העלאת תמונות ציורים
3. עדכון `lib/data.ts` עם נתונים אמיתיים

### שלב 3: דוא"ל (השבוע הבא)
1. הגדרת `Nodemailer`
2. יצירת App Password ב-Gmail
3. עדכון `app/api/orders/route.ts` לשלוח דוא״ל

1. תיקייה Git
2. פתיחת Railway account
3. העלאת הפרויקט
4. כנונפיגורציה של משתנים סביבה
5. בדיקה בייצור

1. בחר מסד נתונים (Supabase מומלץ)
2. מיגרציה של `lib/data.ts` לט"ד
3. בנייה של admin panel מלא

## 📞 בעיות שעלולות להופיע

| בעיה | פתרון |
|-----|------|
| "Cannot find module" | בדוק `yarn install` |
| עמוד לא עדכון | Reload דפדפן (Ctrl+Shift+R) |
| טופס לא שולח | בדוק console (F12) לשגיאות |
| תמונות לא מופיעות | בדוק שהנתיב לתמונה נכון ב-`lib/data.ts` |
| דוא"ל לא עובד | בדוק `.env.local` ו-Gmail settings |

## 💡 הערות חשובות

1. **נתונים בדיקה:** זה "עם תאריך יעילות" - חלף זמן, תצטרך להעלות תמונות אמיתיות וציורים אמיתיים.

2. **דוא"ל:** כרגע הזמנות מאוחסנות בזיכרון בלבד (בטל כשסוגרים את השרת). בדוא״ל הוא קודם.

3. **הגדלה:** כשיהיו יותר ציורים, שקול מסד נתונים למקום `lib/data.ts`.

4. **בטיחות:** אל תשים סיסמאות ישירות בקוד. תמיד השתמש ב-.env.local
